# Boreal Hackathon Submission Notes

## Repository
- GitHub: https://github.com/raldblox/boreal.work
- Live product: https://boreal.work

## Important repo context
This repository is the main Boreal product codebase and includes the hackathon work used for the submission. Boreal is a live request-native commerce product, so the repo also contains broader platform infrastructure beyond the exact Solana hackathon flow. For this submission, judges should focus on the live app in `next-app/`, especially the request-thread product, mounted specialists, and the Solana Operator flow. The current Solana path includes planning-first public execution plus mounted request-thread actions for wallet-approved message signing, memo recording, and simple SOL transfer.

## Suggested access instructions
Open https://boreal.work. The clearest flow is to go to chat, mount `Solana Operator` from Offers, and start a request from there. Some flows require signing in and connecting a Solana wallet because Boreal keeps wallet-approved actions inside the same request thread.

## Demo video outline
Keep the demo under 3 minutes and show the live product, not slides.

1. Open Boreal and state the problem: requests usually fragment across chat, tools, delivery, and proof.
2. Show the Boreal chat surface and mount `Solana Operator` from Offers.
3. Start a request that demonstrates a Solana-native action or planning flow.
4. Show the request thread, team/workboard behavior, and the action path staying attached to the same thread.
5. If wallet flow is available, show wallet-approved signing or memo recording.
6. Close with the key point: one request becomes one tracked execution thread with routing, action, proof, and delivery in one place.

## Pitch video outline
Keep the pitch under 2 minutes.

1. Who we are.
2. What Boreal is: a chat-native market for request-native commerce.
3. Why Solana matters here: low-cost, fast, programmable actions inside a real workflow.
4. Why this team can build it: technical founder with enterprise, blockchain, AI, and systems background.
5. What the wedge is now and what it expands into next.
