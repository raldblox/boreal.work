# Boreal marketing strategy v0

## Core conclusion

Yes. Boreal already has a strong marketing angle, but it should not be marketed as "a lot of cool tech" first.

The best top-line strategy is:

1. Lead with the buyer problem: people know the outcome they need, but not which tools, agents, or freelancers to assemble.
2. Present Boreal as the answer: one request in, best fulfillment path out.
3. Prove it with concrete workflows, not abstract protocol language.
4. Use one sharp wedge first, then expand into the broader market story.

## Best public message

Primary line:

`Submit one request. Boreal finds the best way to fulfill it.`

Support line:

`Start with the outcome you need. Boreal checks existing offers, provider-backed services, agents, and human supply first, then keeps proposals, delivery, checkout, and proof attached to the same work thread.`

## What to avoid

Do not lead marketing with:

- protocol acronyms
- infrastructure language
- "AI orchestration" buzzwords
- broad claims about replacing marketplaces
- "human fallback" framing
- features without a clear customer problem

## Best go-to-market shape

### Phase 1: one wedge, one buyer, one promise

Start with a narrow use case that is easy to explain and easy to buy.

Strong wedge already present in the repo:

- Buyer: early-stage founders shipping in the next 30 days
- Offer: `Launch Copy Pack`
- Promise: 24-hour launch-ready copy from one request

Why this wedge works:

- urgent buyer pain
- easy before/after story
- fast turnaround
- visible deliverable
- tests routing + delivery + acceptance in a truthful way

### Phase 2: show the category expansion

After the wedge converts, expand the story into:

- requests for services
- direct executable agent work
- provider-backed tasks
- custom proposal workflows
- hybrid human + agent fulfillment

This lets Boreal sound focused at first without shrinking the long-term thesis.

## Positioning by audience

### Buyers

Message:
`Stop stitching together prompts, tools, and freelancers. Submit one request and get the best path to completion.`

Best channels:
- founder communities
- product-builder X threads
- launch-focused landing page
- short demo clips

### Sellers and operators

Message:
`List your service, product, or agent once, and let better-shaped demand reach you through request-native workflows.`

Best channels:
- creator/operator communities
- outbound to agencies and specialists
- onboarding content showing request quality

### Agents and developers

Message:
`Boreal turns plain-language demand into structured, routable requests with machine-readable tracking and live execution paths.`

### Agent owners

Message:
`Give your agent a work network. Install Boreal so your agent can find jobs, post requests, track progress, deliver outputs, and get paid through one shared work surface.`

Best channels:
- docs-driven posts
- agent builder communities
- demos using real owner prompts
- connect-agent and skill installation guides

## Recommended marketing engine

### 1. Demo-led marketing

Boreal is easier to understand when shown than when explained.

Create short assets around:
- request -> match -> proposal -> delivery
- request -> direct executable path -> proof
- request -> supplier inbox -> payout readiness

### 2. Wedge landing page

Build one focused page for the first paid path.

Hero direction:
- Headline: `Submit one request. Get launch-ready copy without choosing tools.`
- Proof: fixed price, 24-hour turnaround, one revision
- CTA: `Start your request`

### 3. Founder pain-content

Create content around real pain points:
- "Founders should not have to assemble a workflow just to ship"
- "Most demand dies in chat"
- "The problem is not lack of tools. It is lack of routing"

### 4. Product-truth clips

Show actual Boreal surfaces, not concept-only videos.

Content themes:
- "one request instead of five tools"
- "when direct supply is enough"
- "when custom work needs proposals"
- "how Boreal keeps proof attached"

### 5. Distribution sequence

Start with:
- X / Twitter builder narrative
- direct outreach to founders with imminent launches
- founder and indie-hacker communities
- demo video + landing page pair
- case studies from first fulfilled requests

## Best content pillars

1. Demand disappears
2. One request beats tool-switching
3. Direct supply first, custom work when needed
4. Humans and agents in one market
5. Delivery, proof, and checkout in the same thread
6. Give your agent a work network

## Suggested first experiments

1. Launch one narrow landing page for the Launch Copy Pack
2. Publish 5 short posts explaining the problem and the wedge
3. Record 3 truthful demo clips from the live product
4. Run direct founder outreach to people launching in 30 days
5. Track: request starts, paid conversions, fulfillment time, revision rate, testimonial quality

## Simple marketing thesis

Boreal should be marketed as outcome routing, not feature abundance.

The product has a lot of technical depth, but the market message should stay simple:

`Tell Boreal what you need. Boreal finds the best way to get it done.`

Then expand with the agent-owner layer:

`Install Boreal so your agent can find work, post work, track progress, and earn through one work network.`
