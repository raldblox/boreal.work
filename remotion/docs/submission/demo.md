# Boreal Submission Demo

Use this as the canonical submission demo script.

Purpose:

- live product walkthrough
- not the category pitch

Runtime target:

- `2:00` to `2:30`
- hard cap: `<= 3:00`

## Spoken Script

```text
This demo starts in the public Boreal chat surface with one real request.

I ask Boreal to pressure test a startup idea, define the smallest MVP, and route the right work.

Right away, Boreal is scanning current supply for that request.

You can see matched routes before anything is funded. The owner can take the best route Boreal supports, invite someone into the thread, or open the work to the market.

In this run, I take the top matched route.

Now Boreal turns the ask into a tracked request. This request is the working object. The route, status, approvals, evidence, delivery, and review stay attached here.

Next, the request moves into payment required.

The quote is visible. The seller is visible. And the exact request that will resume after payment is visible too.

Then the owner signs.

Boreal verifies that approval against a real Solana mainnet transaction before work starts.

Once verification lands, the same request resumes. There is no rematch, no second workflow, and no separate checkout path that breaks context.

From here, the specialist works on the same thread. Messages stay attached. Evidence stays attached. Artifacts stay attached. Delivery stays attached. Review stays attached.

Boreal can also support other route types on the same surface. A debate prompt can match to the Debate and Verdict preset team. A broader operations ask can surface current human and agent supply and open to the market when no strong direct lock exists.

The core loop is simple. Ask in chat. Boreal shows matched supply. The owner approves or opens the work. Solana verifies the start. And the same request thread carries the work through fulfillment.

That is Boreal.
```

## Delivery Notes

- Read like a walkthrough, not a trailer.
- Keep the UI actions practical and literal.
- Let the screen do the proving.
- Emphasize `matched supply`, `tracked request`, `funding boundary`, `Solana mainnet transaction`, and `same request resumes`.
